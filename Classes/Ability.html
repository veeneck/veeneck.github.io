Ability

class Ability

Every action, command, spell, buff, etc comes down to an Ability. This class mainly serves as a container for the data while other portions of the code make the magic happen. Let’s go over some terminology which shows the ways an Ability may interact with other portions of the code.

  • Ability: Base class which contains all information about the ability including name, description, cooldown, particles and more.
  • Behavior: Enum describing the different types of abilities offered.
  • Modifier: An actual change an ability may incur. For example, +10 HP or +20 Speed
  • Damage: During combat, a Damage struct is created and passed around. It will contain select information from the ability.
  • Command: An ability on its own will do nothing. A Command is a container for abilities that allows Squads to work with it. A squad will execute a command either instantly, or added to its QueueComponent.
  • CommandInstructions: Abilities have no context. CommandInstructions provide the context by specifying target, location, etc.

  • Action: A wrapper around an ability specificly for UI purposes. Actions will update icons when they are locked, cooling down, etc and give UI feedback.

Creating an Ability
  • init(name:behavior:)

    To create an Ability, add a new entry to the Abilities.pList file. You can see other entries for reference, or look at the member variables of this class to use as keys. When adding an entry, a Behavior has to be selected. Also, consider what sort of Modifier to add as Instant, Spell, and potentially other abilities may do nothing without a modifier.

    Declaration

    Swift

    init(name:String, behavior:Ability.Behavior)

Behaviors
  • Behavior

    Hard coded options for how an ability can behave. Used across the entire code base.

    • Move: Travel to valid lcoation.
    • Melee: Travel to engage range and attack.
    • Ranged: Attack immediately if target in range.
    • Instant: Immediately process ability when activated.
    • Spell: Combine animation with immediate action.
    See more

    Declaration

    Swift

    enum Behavior : String

Validation
  • Validation

    Ability can only be used if the Validation criteria is met.

    • Moving: Squad is moving.
    • NotMoving: Squad isn’t moving.
    • InCombat: Squad is in combat.
    • NotInCombat: Squad is not in combat.
    See more
