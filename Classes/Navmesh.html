Navmesh

Extension of GKObstacleGraph that has built in utility methods for pathfinding. For example, if the desired point is inside of an obstacle, you can optionally find the closest point on the graph.

Initializing a Navmesh
  • init(file:scene:bufferRadius:)

    Reads a PList file with details on the obstacles. Creates a sprite for each obstacle, with a CGPath representing the data. Path is converted to a physics body, which is used for the obstacle in the GKGraph.

Pathfinding
  • findPath(_:end:ignoringObstacles:radius:scene:)

    Attempts to find a path through the graph while ignoring any obstacles you wish. Obstacles apply to the placement of the start and end node (i.e.: a unit lies inside of an obstacle and needs to get out).
