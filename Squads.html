  • DefendingState

    The purpose of this state is to wait while being attacked by a Melee unit for the first time. When engaged, this state will trigger, which causes the defender to stop moving and wait. The attacker will seek, and once the attacker has reached the defender, the defender can then switch to a normal combat state. This is to allow things like charge, unit targeting, etc to work reliably. If a unit is already engaged in Melee combat, it would not enter this state should another attacker join the fight.
